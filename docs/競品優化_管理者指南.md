## 1. 權限說明

系統依 Google 帳號 Email 控管頁籤存取，若看不到某個頁籤，請聯繫管理員申請權限。

### 申請權限時，請提供以下資訊

| 欄位 | 說明 |
|------|------|
| Email | 登入系統使用的公司 Google 帳號 |
| 單位 | 所屬部門或團隊名稱 |
| 可查看頁籤 | 需要存取的頁籤名稱（見下方清單） |

### 可申請的頁籤清單

| 頁籤名稱 | 內容說明 |
|---------|---------|
| 星城 / 滿貫大亨 / 金好運 … 等各競品時間軸 | 各競品活動甘特圖，可個別申請特定競品 |
| 跨產品搜尋 | 同時搜尋所有競品活動 |
| 活動類型總表 | 各競品遊戲項目分類整理 |
| 行銷資料庫 | 競品行銷活動甘特圖 |
| 數學資料庫 | 競品老虎機遊戲規格 |
| 營收分析 | SensorTower 競品 App 營收與下載量 |
| 其他資料 | 競品相關連結與營運簡報 |

> 💡 如需一次開放所有頁籤，申請時說明「全部頁籤」即可，管理員會統一設定。

## 3. 快取更新（管理員操作）

> 🎬 操作教學影片：`參考影片/快取更新教學_h264.mp4`（HTML 版本內嵌於本節上方）

系統資料來自 Google Sheets 快取，需【**手動**】觸發更新：

**步驟**：

1. 開啟 [競品追蹤 Google Sheets](https://docs.google.com/spreadsheets/d/1TD-uKw3pz9ax7CW2WgtS8z4xfal-kCODliT4PU2CqoI/edit?gid=796982050)
2. 點選選單「資料管理」→「更新活動快取」
3. 選擇要更新的**年度**（支援多選，可同時更新多個年度）
4. 選擇**部門**（企劃 / 行銷 / 數學）
5. 點選「開始更新」
6. 系統會自動掃描對應年度的 Google Slides，**若資料量大（超過 4 分鐘），會自動分段續傳**，過程中請勿關閉瀏覽器

**更新進度確認**：可從同一選單點選「檢查進度」查看目前掃描到第幾個年度/產品。

**更新頻率建議**：每季簡報完成後更新一次。

---

### 【兩階段寫入說明】

快取更新採用**兩階段流程**，確保資料不會在未確認前影響系統：

| 階段 | 工作表 | 說明 |
|------|--------|------|
| 第一階段（自動） | `投影片資料快取` | 腳本掃描完畢後，所有資料先寫入此分頁 |
| 第二階段（手動確認） | `企劃資料快取` / `行銷html資料快取` / `數學html資料快取` | 確認資料無誤後，手動貼至對應部門快取，Web App 才會讀到新資料 |

> ⚠️ 若只完成第一階段而未套用至部門快取，系統前端顯示的資料不會改變，建議第一階段要移動至第二階段時，先對第二階段進行備份，以防資料遺失。

> 💡 **執行第二階段前，建議先確認異常 LOG（見下方說明）**，若有資料問題請先修正來源簡報並重新執行快取更新，確認無誤後再套用至部門快取。

---

### 【異常 LOG 說明】

> ⚠️ 更新完成後，系統若發現問題會自動在 Google Sheets 建立「**異常LOG**」工作表，記錄兩類狀況：

---

#### 一、跨年活動（最常見）

活動日期橫跨兩個年度時（如 2025/12/20 ~ 2026/1/5），系統的處理方式：

1. **自動擴展掃描範圍**：選 2025 時，系統會自動同時掃描 2024、2025、2026 的資料夾，確保跨年活動不漏掉
2. **自動拆分寫入快取**：2025/12/20~12/31 寫入 2025 快取，2026/1/1~1/5 寫入 2026 快取
3. **記錄到 LOG**：純紀錄用途，**不需要任何手動操作**

| 欄位 | 說明 |
|------|------|
| 時間戳記 | 本次更新的執行時間 |
| 產品名稱 | 哪個競品的活動 |
| 掃描年度 | 本次掃描的年度 |
| 跨年年度 | 活動實際延伸到哪個年度 |
| 活動數量 | 受影響的活動筆數 |
| 活動清單 | 活動名稱、類型、日期範圍（逐筆列出） |
| 狀態 | 系統自動填入「待處理」，純紀錄用，不需要手動修改 |

> 💡 看到 LOG 有跨年活動不需要緊張，代表系統已正常處理完畢。

---

#### 二、讀取錯誤

掃描 Google Slides 時若遇到無法解析的內容，會記錄到同一個「異常LOG」分頁，格式不同（無「跨年年度」欄）：

| 錯誤類型 | 意思 | 處理建議 |
|----------|------|----------|
| 簡報讀取失敗 | 某份 Slides 無法開啟，通常是權限問題 | 確認該 Slides 已開放給腳本帳號存取 |
| 表格讀取失敗 | 某張投影片的表格結構無法解析 | 開啟來源欄的連結，檢查該頁表格格式 |
| Shape讀取失敗 | 投影片內某個圖形物件讀取失敗 | 通常可忽略（非表格內容） |
| 日期解析錯誤 / 日期格式錯誤 | 活動日期欄填寫格式不符（如少了斜線、多了空格） | 開啟來源連結，修正該格日期格式後重新更新 |
| 缺少分類 | 活動內容無法對應任何活動類型 | 確認該活動是否歸類正確，或新增對應的活動類型 |
| 分類 / 內容讀取錯誤 | 解析表格列時發生例外 | 查看「錯誤詳情」欄的具體訊息 |

**查看方式**：更新完成後開啟競品追蹤 Sheets，點選底部「**異常LOG**」分頁（若無此分頁表示本次更新沒有任何異常）。

> ⚠️ 修復問題簡報後，需重新執行一次「快取更新」，才能讓修正後的內容寫入快取。異常 LOG 本身不會自動清除，確認處理完畢後可手動刪除該列。

## 4. 常見問題

### Q: 程式碼更新後，該怎麼重新部署？網址會變嗎？

GAS Web App 有兩種部署方式，效果不同：

| 操作 | 路徑 | 網址 | 適用情況 |
|------|------|------|---------|
| **更新現有部署** | 部署 → 管理部署 → 編輯（鉛筆圖示）→ 版本選「新版本」→ 部署 | **不變** | 所有程式碼更新，不論大小，包含功能改寫、全新版本 |
| **新增部署** | 部署 → 新增部署 | **產生新網址** | ① 需要改「執行身分」或「存取權限」設定　② 想讓新舊版本同時並存（如測試用）　③ 舊部署損壞或封存後需重新建立 |

程式碼更新不管改多大，都只需要「更新現有部署」，網址不會變。「新增部署」不是為了大版本更新用的，是為了改部署設定或建立獨立環境。

**如果網址換了，舊網址會怎樣？**

舊網址仍然有效，但指向**舊版程式碼**，使用者還在舊網址上會看到更新前的畫面。需要手動到「管理部署」將舊部署封存（Archive）才會停用。

處理步驟：
1. 確認新網址運作正常
2. 到「管理部署」將舊部署封存
3. 更新所有已分享的連結（書籤、內部文件、使用指南等）
4. 通知使用者更換網址

### Q: 如何新增競品的 SensorTower 營收查詢？

需修改後端程式碼 `競品圖表.txt` 的 `SENSOR_TOWER_CONFIG`，加入新競品的 Product ID：

> ⚠️ 通常會找數據協助

```javascript
PRODUCT_IDS: [
  // 現有競品...
  'xxxxxxxxxxxxxxxxxxxxxxxx'   // 新競品 Product ID
],
PRODUCT_NAMES: {
  // 現有競品...
  'xxxxxxxxxxxxxxxxxxxxxxxx': '新競品名稱'
}
```

同時在前端 `util.html` 的 `SENSOR_TOWER_CONFIG.PRODUCT_NAMES` 陣列補上名稱：

```javascript
PRODUCT_NAMES: ['包你發', '星城', ..., '新競品名稱']
```

> ⚠️ Product ID 需從 SensorTower 平台查詢，修改後需重新部署 GAS Web App 才會生效。
