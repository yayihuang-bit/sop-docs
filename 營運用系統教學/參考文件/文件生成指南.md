## 🏃 快速入門

製作Google Docs範本後→透過「[營運自動化文件表](https://docs.google.com/spreadsheets/d/1uIYYoJn2iYkWIx9oZLdABD2x1kUtnI8dEem9Uv0QbGc/edit?gid=1716214522#gid=1716214522)」把活動資料套入 Google Docs 範本，批次生成文案文件，再推進[自動發布系統](自動發布指南.html)。

### 已有固定範本時，後續流程

```
C欄輸入活動名稱
(智慧搜尋套入範本)
         ↓
填入必填欄位
(標題、內容、活動辦法)
         ↓
D欄打 OK
         ↓
生成營運文案
(文件存入 Drive)
         ↓
確認完整
(推進 auto-post)
```

## 一、操作步驟
<video controls src="../示意圖/文件生成流程.mp4" title="Title"></video>

1. 開啟「[營運自動化文件表](https://docs.google.com/spreadsheets/d/1uIYYoJn2iYkWIx9oZLdABD2x1kUtnI8dEem9Uv0QbGc/edit?gid=1716214522#gid=1716214522)」，進入「在這生成文件」分頁

2. 在 **C 欄**輸入活動名稱關鍵字
   - 系統自動比對範本庫，套入對應整列資料
   - 若有多個候選 → 格子變黃底出現下拉選單，選擇正確版本

3. 填入欄位：**黃色底** = 必填 ，**白色** = 視需求修改 ，**綠色** = 有自動化設定功能 ，**灰色** = 不用改

4. **D 欄**填入 `OK`，標記此列準備生成

5. 點上方選單「⭐ 自動化工具 ⭐」→「生成營運文案」

6. 系統生成文件、存入 [Drive 資料夾](https://drive.google.com/drive/folders/1qdBRQJ0M3rnOTVYoIuK2wYttMiM5jLF1)，A 欄自動標記 `DONE`

7. 等活動資料確認完整後，把文件連結以**超連結方式**貼進 [營運文件試算表](https://docs.google.com/spreadsheets/d/1qmRZ-OT-PW6IzQFoJatZKf8fMUwnnBX-gyYHZCxuL1E/edit?gid=1234248588#gid=1234248588) E 欄，F 欄改為「待設定」

> 💡 生成的文件連結可在試算表 D 欄的「生成後文件放這點此」找到，或直接去 Google Drive 對應資料夾。

## 二、範本製作

範本是一份 Google Docs 文件，內容由**固定文字**和 **`{{變數}}`** 組成。生成時系統把試算表那列的值替換進去。

> 📁 所有自動化範本放這：[範本資料夾](https://drive.google.com/drive/folders/1wgb4AfCGnxjrDIBvfQNixDz-nRXHIZm3)

### 2.1 變數對照規則

試算表**第 1 列的欄位標題**，就是 Docs 範本裡的 placeholder 名稱，兩邊完全一致才能替換。

| 試算表第 1 列標題 | Docs 範本裡寫 | 生成後替換成（範例） |
|-----------------|-------------|-----------------|
| `{{主標}}` | `{{主標}}` | 香包大集合 |
| `{{開始日}}` | `{{開始日}}` | 2026/6/15 |
| `{{開始時間}}` | `{{開始時間}}` | 19:00:00 |
| `{{結束日}}` | `{{結束日}}` | 2026/6/22 |
| `{{文件年月份}}` | `{{文件年月份}}` | 2026年6月 |

多個變數可以組合在同一行，例如：

| Docs 範本裡寫 | 生成後替換成 |
|-------------|-----------|
| `活動期間：{{開始日}} {{開始時間}} - {{結束日}} {{結束時間}}` | 活動期間：2026/6/15 19:00:00 - 2026/6/22 22:00:00 |

> 💡 可用的變數清單 = 試算表第 1 列所有 `{{...}}` 格式的欄位標題。新增欄位就等於新增可用變數。

### 2.2 公式欄位

試算表某些欄位不是手動填，而是用公式自動計算。例如：

- 活動結束日 + 3 天 = icon 移除日 → 欄位公式寫 `=結束日欄位+3`
- 這類欄位由範本製作者在試算表設好，使用者只需填入基本日期，其他衍生日期自動推算

> 💡 生成時腳本讀取欄位的「顯示值」（計算結果），不是公式本身。生成出來的文件會是實際日期，不會出現 `=K3+3`。

### 2.3 區段格式（配合 auto-post）

若要讓 auto-post 腳本自動讀取，區段標題需符合以下格式（`🤖` 開頭代表自動化）：

| Docs 裡的區段標題 | 對應腳本 |
|-----------------|--------|
| `🤖官網設定-文章管理-AI自動化用` | 官網-文章 |
| `🤖官網設定-廣告設定` | 官網-廣告設定 |
| `🤖APP-公告系統-跑馬燈設定` | App-跑馬燈設定 |
| `🤖APP-客服系統-定時廣播` | App-定時廣播 |

區段內欄位格式：`欄位名稱：{{變數}}`，例如：

```
標題：{{主標}}！{{副標}}
上架時間：{{文章上架日}} {{上架時間}}
```

> ⚠️ 欄位名稱（如「標題：」「上架時間：」）需與 auto-post 腳本讀取的名稱完全一致，有異動需同步修改腳本。
