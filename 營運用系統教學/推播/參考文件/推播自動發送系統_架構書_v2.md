# 推播自動發送系統 — 架構書 v2.0
> 本文件描述系統完整需求、操作情境與技術架構，供維護者與接手人員參考。

---

## 一、專案目標

建立一套讓**非技術人員**也能操作的推播自動發送系統。

**操作者只需要做兩件事：登入 Google 帳號與後台系統後，在 Google Sheets (https://reurl.cc/lpbgOd)填入推播資料，再透過排程設定讓系統自動執行發送。**

系統負責：
- 自動讀取試算表中標記為「待發送」的資料
- 驗證每筆資料是否完整（日期、時間、標題、廣告內容）
- 依序填入後台推播表單並送出
- 送出成功後自動將試算表狀態更新為「已發送」，防止重複發送
- 自動從後台匯出的 xlsx 讀取推播統計數據，回填至試算表 H / I / J 欄

## 二、系統架構（三層）

```
第零層：操作者入口
  網頁介面（http://localhost:3000）
  → 推播發送 / 統計回填 / 排程設定 / 帳號管理 / Google 帳號切換
         ↓
第一層：資料來源 https://reurl.cc/lpbgOd
  A. Google Sheets（🤖推播資料彙整 分頁）
     → A欄：上架日期 / B欄：時間 / C欄：標題 / D欄：廣告內容
       G欄：狀態 / H欄：點閱次數 / I欄：發送次數 / J欄：開啟率
  B. PushNotification*.xlsx（從後台推播設定頁匯出）
     → 日期 / 時間 / 標題 / 內容 / 點閱次數 / 發送次數 / 開啟率
         ↓
第二層：自動化執行
  Playwright（使用系統 Chrome，channel: 'chrome'）
  → 用已存的 Google 登入狀態讀取試算表 CSV
  → 登入後台填寫推播表單並送出
  → 操作 Google Sheets UI 更新 G / H / I / J 欄
```

**操作者只操作第零層，其餘全部自動執行。**

## 三、操作情境

### 情境一：日常發送推播

```
操作者在 Google Sheets 填好推播資料：
  日期：2026/4/10
  時間：19:01:00
  標題：只升一等就有紅卡？
  廣告內容：天天衝等！戰國霸主紅卡送乎你！
  狀態：待發送

→ 點兩下桌面的 start.bat
→ 瀏覽器自動打開介面
→ 點「立即執行發送」

系統自動：
  ✅ 讀取所有「待發送」資料
  ✅ 逐筆填入後台推播表單
  ✅ 送出確認
  ✅ 將試算表狀態更新為「已發送」
```

### 情境二：設定每天自動排程

```
操作者進入介面 → 排程設定 → 輸入時間（例如 10:00）→ 儲存

系統在該小時內自動執行一次（當天只執行一次），
依序執行：推播發送完成後，等待 8 秒，再自動接續統計回填。
（等待原因：兩個腳本共用同一個 Chrome profile，需等前一個完全釋放）

注意：需保持 start.bat 在背景執行中（cmd 視窗不可關閉）。
注意：若要重新啟動，需先關閉舊的 cmd 視窗，再點 start.bat。
```

### 情境三：統計數據回填

```
操作者進入介面 → 點「匯入統計數據」

系統自動：
  ✅ 登入後台，前往「推播設定」頁面
  ✅ 全選並匯出推播資料（自動下載 xlsx，不需手動操作）
  ✅ 讀取 xlsx（只處理 30 天內的資料）
  ✅ 比對試算表中的日期 + 時間
  ✅ 將 H欄（點閱次數）、I欄（發送次數）、J欄（開啟率）寫入（12pt）
  ⚠️ 若有 xlsx 資料在試算表找不到對應列，彈出視窗通知操作者
```

### 情境四：切換 Google 帳號

```
操作者進入介面 → Google 帳號設定 → 點「重新登入 Google 帳號」

瀏覽器自動開啟 → 完成 Google 登入 → 關閉瀏覽器

下次執行自動使用新帳號的試算表存取權限。
```

### 情境五：欄位不完整的資料被跳過

```
試算表某列：
  日期：2026/4/11
  時間：（空白）
  標題：抽獎券最後倒數啦～
  廣告內容：頭像框！努力士、黃金，一個都別漏掉！
  狀態：待發送

→ 系統偵測到 B欄（時間）空白
→ 跳過此列，印出警告：⏭️ 跳過 (欄位不完整): 2026/4/11 抽獎券最後倒數啦～
→ 繼續處理下一筆
```

### 情境六：更換後台帳號密碼

```
操作者進入介面 → 後台帳號設定 → 修改帳號/密碼 → 儲存

設定即時生效，下次執行自動使用新帳密。
```

## 四、技術選型

| 元件 | 工具 | 說明 |
|---|---|---|
| 網頁介面 | Node.js `http` 模組 | 輕量 Web Server，無需安裝額外框架 |
| 瀏覽器自動化 | Playwright (系統 Chrome) | 使用 `channel: 'chrome'` 直接呼叫已安裝的 Chrome，無需額外下載瀏覽器 |
| 試算表讀取 | Google Sheets CSV Export | 透過 Playwright 下載 CSV，不需要 OAuth |
| 試算表寫入 | Google Sheets UI 操作 | 用 Playwright 操作 Name Box + Formula Bar 直接寫入 |
| xlsx 解析 | `xlsx` npm 套件 | 讀取後台匯出的 PushNotification*.xlsx |
| Google 登入狀態 | Playwright PersistentContext | 儲存在 `google_auth/` 資料夾，永久保留 |
| 設定儲存 | `config.json` | 帳號、密碼、排程時間本地儲存 |
| 排程 | `setInterval` 每分鐘比對小時 | 設定小時觸發，當天只執行一次，不依賴 Windows 工作排程器 |
| 前端 Log 串流 | Server-Sent Events (SSE) | 執行過程即時顯示在介面上 |
| 未配對通知 | Modal 彈出視窗 | 統計回填時若有資料找不到對應列，立即通知操作者 |

## 六、資料流程

### 推播發送流程

```
1. 使用者點「立即執行發送」或排程時間到達
       ↓
2. ui-server.js 呼叫 auto-post-from-sheet.js
       ↓
3. Playwright 開啟系統 Chrome（使用 google_auth/ 登入狀態）
       ↓
4. 下載試算表 CSV（Google Sheets Export URL）
       ↓
5. 解析 CSV，找出「待發送」且 ABCD 完整的列
       ↓
6. 登入後台（backstage.online808.com）
       ↓
7. 逐筆：填寫開始時間 → 敘述 → 廣告內容 → 點確定新增 → 確認彈窗
       ↓
8. 發送成功 → 開啟試算表 → 找到對應列 → G欄寫入「已發送」
       ↓
9. 執行完畢，回報介面顯示結果
```

### 統計回填流程

```
1. 使用者點「匯入統計數據」或排程時間到達
       ↓
2. ui-server.js 呼叫 sync-push-stats.js
       ↓
3. Playwright 登入後台，前往「推播設定」頁面
       ↓
4. 全選推播列表 → 點「匯出資料」→ 自動下載 xlsx
       ↓
5. 解析 xlsx，只保留 30 天內的資料
   建立查找表：「日期_時間」→ { 點閱次數, 發送次數, 開啟率 }
       ↓
6. 下載試算表 CSV，比對每列的日期 + 時間
       ↓
7. 找到對應列 → 開啟 Google Sheets
       ↓
8. 逐列寫入 H欄（點閱次數）、I欄（發送次數）、J欄（開啟率），字體 12pt
       ↓
9. 若有 xlsx 資料找不到對應列 → 介面彈出 Modal 通知未配對清單
```

## 七、防呆機制

| 情況 | 處理方式 |
|---|---|
| ABCD 任一欄位空白 | 跳過該列，印出警告，不發送 |
| G 欄已是「已發送」 | 跳過，不重複發送 |
| G 欄不是「待發送」 | 跳過（只處理明確標記的列） |
| 後台登入失敗 | 繼續嘗試，印出錯誤訊息 |
| 試算表行號偏移 | 動態定位 header 列，自動校正行號 |
| xlsx 超過 30 天的資料 | 自動排除，不寫入試算表 |
| xlsx 資料在試算表找不到對應列 | 彈出 Modal 視窗列出未配對清單 |
| Port 3000 已被佔用 | 需先關閉舊的 cmd 視窗再啟動 |
| 排程重複觸發 | `scheduleFiredDate` 確保每天只執行一次 |
| 統計回填連線後台失敗（網路異常） | 印出錯誤訊息後正常結束，不 crash 整個 Node process |
| 排程連續執行兩腳本時 Chrome profile 衝突 | 推播發送結束後等待 8 秒再啟動統計回填 |
