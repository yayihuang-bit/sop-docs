## 1. 系統概述

本系統是一套部署於 Google Apps Script (GAS) 平台的**競品活動分析 Web App**，提供台灣手機遊戲市場（老虎機／麻將類）的競品追蹤、活動時間軸視覺化、營收分析與跨產品搜尋功能。

### 追蹤競品

星城、滿貫大亨、金好運、豪神、HUGA、錢街、老子有錢、寶島、嚦咕麻將、王牌、大頭家

### 核心資料來源

| 來源 | 說明 | 實際位置 |
|------|------|----------|
| Google Sheets（主試算表） | 快取資料儲存、系統輸出目標 | [試算表](https://docs.google.com/spreadsheets/d/1TD-uKw3pz9ax7CW2WgtS8z4xfal-kCODliT4PU2CqoI) |
| └ 目錄工作表 | 其他資料頁籤的連結來源（需手動維護） | [目錄工作表](https://docs.google.com/spreadsheets/d/1TD-uKw3pz9ax7CW2WgtS8z4xfal-kCODliT4PU2CqoI/edit?gid=796982050) |

## 6. 後端模組詳解

### 6.1 主後端（競品圖表.txt）

#### 6.1.1 Web App 入口

```
doGet()
  └── HtmlService.createTemplateFromFile('圖表').evaluate()
        └── include(filename) 動態拼接各 HTML 片段
```

#### 6.1.2 競品資料讀取流程

```
getAllCompetitorStats()
  ├── [快取存在] readFromCache(cacheSheet)
  │     └── 讀取「企劃資料快取」A-E 欄 JSON → 去重 → 回傳
  └── [快取不存在] getAllCompetitorStatsRealtime(spreadsheet)
        └── 掃描 {競品名}時間軸 工作表
              └── processCompetitorSheetEnhanced(sheet)
                    ├── 解析月份標題（年月格式）
                    ├── 解析活動類型列
                    ├── 解析活動內容（含跨月/跨年）
                    └── 回傳 { stats: [...], activities: [...] }
```

#### 6.1.3 SensorTower 代理

```
getSensorTowerData(startDate, endDate, selectedProductNames)
  ├── 白名單驗證：將產品名稱對照後端 PRODUCT_NAMES，過濾非法名稱
  ├── 名稱→ID 轉換：後端自行查找，Product ID 不暴露至前端
  └── UrlFetchApp.fetch(SensorTower API URL)
        ├── AUTH_TOKEN（從 Script Properties 讀取）
        ├── COUNTRY: TW
        └── 回傳 { date, app_id, android_revenue, iphone_revenue, ... }
```

前端傳入**產品名稱**（如「星城」），後端自行轉換為 Product ID 後呼叫 API。Product ID 完全不出現在前端程式碼中。

#### 6.1.4 快取讀取與去重

快取工作表欄位結構（A-E 欄為企劃、K-O 欄為行銷）：

| 欄 | 內容 |
|----|------|
| A / K | 競品名稱 |
| B / L | 季度（格式：`2025-Q3`） |
| C / M | 類型（固定為 `all`） |
| D / N | JSON 資料（含 `stats[]` 與 `activities[]`） |
| E / O | 更新時間戳記 |

去重邏輯：以活動名稱（`activity.name`）為 key，快取讀取後執行 `removeDuplicateActivities()` 確保不重複。

#### 6.1.5 權限系統

```
getUserPermissions(userEmail)
  └── 查詢「權限管理」工作表
        ├── 欄位：信箱 / 可查看頁籤(每個項目請用,分開)
        └── parsePermissions(permissionStr)
              ├── 'all' / 'admin' → 所有頁籤
              ├── 'basic' → 營收分析、活動類型總表、其他資料
              └── 個別頁籤名稱直接授權
```

**重要限制**：後端在 `getAllCompetitorStats()` 中依 `allowedTabs` 過濾競品資料，但 `getMathDatabaseData()`、`getMarketingDatabaseData()` 等函數在後端以 `callerHasTab_()` 驗證，若無對應權限回傳空陣列。

### 6.2 快取建置引擎（自動刷新時間軸.txt）

#### 6.2.1 觸發方式

- **手動**：Google Sheets 選單 → 資料管理 → 更新快取
- **自動續傳**：當主流程執行超過 4 分鐘（GAS 限制），自動建立「30 秒後觸發一次」的時間觸發器

#### 6.2.2 主流程

```
showYearPickerForCache()
  └── 顯示年度 + 部門選擇對話框
        └── startCacheUpdateWithYears(selectedYears, selectedDepts)
              └── directExtractToCache()  ← 核心主流程（含斷點續傳）
                    ├── [第一次] 清空快取工作表
                    ├── 讀取斷點（資料暫存_快取 A2:D2）
                    ├── 逐年逐產品掃描 Drive 資料夾
                    │     └── processFolder()
                    │           ├── extractActivities()  解析時間軸表格
                    │           └── extractMathTimeline() 解析數學表格
                    ├── filterDuplicateActivities()  去重
                    ├── writeToCacheDirectly()  寫入企劃/行銷快取
                    ├── writeToMathCache()  寫入數學快取
                    └── [4 分鐘到] 建立 30 秒後觸發器 → 記錄斷點 → 結束
```

## 6.3 投影片表格解析邏輯（extractActivities / extractMathTimeline）

### 6.3.1 Drive 資料夾結構與掃描規則

系統掃描三個根資料夾，每個資料夾的目錄結構相同：

```
{ROOT / SLOT / MATH}_FOLDER_ID/
  ├── 2025/                       ← 年度資料夾（符合 /^\d{4}$/ 的資料夾名稱）
  │     └── {競品名}/             ← 競品子資料夾
  │           └── {檔名}營運簡報.gslides   ← 檔名需包含「營運簡報」才會被處理
  └── 歷屆資料/
        └── 2024/
              └── {競品名}/
                    └── ...
```

> ⚠️ 只有檔名包含「**營運簡報**」的 Google Slides 才會被開啟解析，其他檔案一律略過。

---

### 6.3.2 投影片中的時間軸偵測

`processFolder()` 對每張投影片（slide）執行兩道掃描，找到時間軸後才呼叫 `extractActivities()`：

**第一道：Shape 文字掃描**

```
for each shape on slide:
  if shape.text.includes('時間軸') AND NOT includes('數學時間軸'):
    productName = shape.text.trim()
    extractActivities(tables[0], ...)   ← 取該頁第一個表格
```

**第二道：表格首欄掃描**

```
for each table on slide:
  if table.cell(0, 0).includes('時間軸') AND NOT includes('數學時間軸'):
    productName = cell(0, 0).text.trim()
    extractActivities(table, ...)
```

**競品名稱清理**：取得 productName 後，在寫入快取前執行：

```javascript
cleanProductName = productName.replace('時間軸', '').replace('行銷', '').trim()
// 例：「星城時間軸」→「星城」、「星城行銷時間軸」→「星城」
```

**範例表格跳過**：`extractActivities()` 入口處掃描所有列的第一欄（col 0），若任一列包含「**範例**」，整張表格略過不解析。

---

### 6.3.3 活動時間軸表格欄位結構

```
┌──────┬──────────────┬────────────────────────────────────────────────┐
│ 列 0 │  (標題列)     │  ← 跳過（i 從 1 開始）                         │
│ 列 1 │ col1: 類型   │ col2: 活動內容（多行文字，一行一個活動）         │
│ 列 2 │ col1: 空白   │ col2: 活動內容（繼承上一列的類型）               │
│ 列 3 │ col1: 新類型 │ col2: 活動內容                                   │
│  ... │  ...          │  ...                                            │
└──────┴──────────────┴────────────────────────────────────────────────┘
```

| 欄位 | 欄號（0-indexed） | 說明 |
|------|-----------------|------|
| 活動類型 | col 1（B 欄） | 可為空白，空白時沿用上一列的類型（`currentType`） |
| 活動內容 | col 2（C 欄） | 多行文字，每行一筆活動；若為 `-` 則跳過 |

> col 0（A 欄）在 `extractActivities()` 中只用於「範例偵測」，不讀取業務資料。

---

### 6.3.4 活動文字格式與日期解析

每行活動文字以以下正規式解析：

```
/^(\[.*?\])?\s*(.+?)\s*\((\d{4}\/)?(\d{1,2}\/\d{1,2})(?:~|-|～)?(?:(\d{4}\/)?(\d{1,2}\/\d{1,2}))?\)/
```

| 群組 | 對應 | 說明 |
|------|------|------|
| `(\[.*?\])?` | 前綴標籤（可選） | 如 `[排行榜]`、`[新海皇]` |
| `(.+?)` | 活動名稱 | |
| `(\d{4}\/)?` | 開始年份（可選） | 如 `2025/`；省略時使用資料夾年度 |
| `(\d{1,2}\/\d{1,2})` | 開始日期 | `MM/DD` 格式，必填 |
| `(?:~\|-\|～)?` | 分隔符（可選） | `~`、`-`、`～` 皆可 |
| `(?:(\d{4}\/)?(\d{1,2}\/\d{1,2}))?` | 結束年份＋日期（可選） | 省略時同開始日 |

**合法格式範例**：

```
排行榜活動(12/1~12/31)
[排行榜] 週末排行榜(12/28~1/5)          ← 跨年，自動偵測
特別活動(2025/11/25~2025/12/31)          ← 明確指定年份
單日活動(3/15)                           ← 無結束日，視為同一天
```

**跨年自動偵測**（結束年份省略時）：

```javascript
if (endMonth < startMonth) {
  endYear = startYear + 1   // 結束月份 < 開始月份 → 視為跨年
}
```

**日期解析失敗的錯誤分支**：

| 情境 | LOG 類型 |
|------|----------|
| 括號內有數字但格式非標準（如「售完為止」） | `日期格式異常` |
| 括號內完全沒有數字 | `日期格式錯誤` |
| `new Date()` 拋出例外 | `日期解析錯誤` |

### 6.3.5 活動類型細分規則

解析完成後，`currentType` 會再經過細分函數處理：

#### 魚機細分（`getFishGameSubcategory()`）

取活動名稱的 `[前綴]` 做字串前綴比對：

| 前綴條件 | 細分類型 |
|----------|---------|
| 以 `新海皇` 或 `新魚種` 開頭 | `魚機-新海皇/魚種` |
| 包含 `活動海皇` | `魚機-活動海皇` |
| 以 `主特色改版` 或 `主特色` 開頭 | `魚機-主特色改版` |
| 包含 `優化` | `魚機-優化` |
| 包含 `排行榜`、`返水`、`外贈` | `魚機-活動` |
| 其他 | `魚機-活動` |

#### 其他類型細分（`getOtherTypeSubcategory()`，僅限企劃簡報）

取 `[前綴]`，以 **Levenshtein 相似度 ≥ 0.5** 比對以下關鍵字群：

| 關鍵字群 | 細分類型 |
|----------|---------|
| `新系統功能`、`新系統`、`系統功能`、`新功能` | `新系統功能` |
| `儲值`、`充值`、`儲金` | `儲值` |
| `每登`、`登入` | `每登` |
| `CCU`、`人數`、`在線` | `CCU` |
| `優化`、`改版`、`修正` | `其他-優化` |
| 無匹配 | `其他` |

> ⚠️ 行銷簡報來源（`source === '行銷簡報'`）不做其他類型細分，直接保留 `其他類型`。

### 6.3.6 去重邏輯（`filterDuplicateActivities()`）

同一競品所有活動合併後執行去重，去重 key 為：

```javascript
key = `${type}_${normalizedName}_${startDate.getTime()}_${endDate.getTime()}`

其中 `normalizedName` 將多餘空白壓縮為單一空格，並統一 `～` / `-` 為 `~`。

## 附錄 A：簡報範本插入工具（Insert Proposal Slides）

> ⚠️ 此工具與競品分析 Web App **完全獨立**，不共用任何資料來源、快取或部署環境。

### A.1 系統概述

`Insert Proposal Slides.txt` 是一套直接附加在 **Google Slides** 上的 GAS Container-bound Script（非 Web App）。使用者可在任何簡報中開啟側邊欄，從統一的範本庫挑選投影片並插入到目前選取頁面後方。

| 項目 | 說明 |
|------|------|
| 載體 | Google Slides（Container-bound Script） |
| 入口 | `onOpen()` → 選單「⭐開啟範本工具⭐」→「顯示範本選單」 |
| 範本來源 | 固定一份 Google Slides（`TEMPLATE_PRESENTATION_ID`） |
| 快取機制 | `ScriptProperties`（腳本層級，所有使用者共用） |
| 主題偏好 | `UserProperties`（個人層級，跨裝置同步） |

---

### A.2 範本庫結構

所有範本投影片集中存放於 `TEMPLATE_PRESENTATION_ID` 指向的 Google Slides。每個範本區塊的**第一張投影片**左側需有一個文字框（`shape.getLeft() < 100`），格式如下：

```
主項：XXX
子項：XXX
細項：XXX
```

- 沒有標記的後續投影片自動歸入前一個範本區塊
- 側邊欄依三層結構顯示選單：**主項 → 子項 → 細項**（點擊細項即插入）

### A.3 主要函數

| 函數 | 說明 |
|------|------|
| `onOpen()` | 簡報開啟時自動建立選單 |
| `createSidebar()` | 讀取分類資料、套用主題、渲染側邊欄 HTML |
| `getTemplateCategories()` | 快取優先讀取分類，有變更才觸發重建 |

| `applyTemplate(type)` | 將範本投影片插入目前選取頁後方 |
| `checkTemplateUpdate()` | 比對快取時間與檔案修改時間，回傳是否需要更新 |
| `refreshSidebar()` | 強制重建索引並重新開啟側邊欄 |

### A.4 快取機制（ScriptProperties）

| 鍵值 | 內容 |
|------|------|
| `TEMPLATE_INDEX` | 三層分類結構 JSON |
| `TEMPLATE_MODIFIED_TIME` | 範本簡報最後修改時間（毫秒字串） |

```

  ├── 快取存在 且 時間戳記相符 → 直接回傳 JSON.parse(cached)
  └── 快取不存在 或 已過期    → rebuildIndex()
        ├── 掃描範本簡報所有投影片
        ├── 找到含「主項：」的標記框 → 記錄 main/sub/detail + 投影片 index
        ├── 無標記投影片 → 歸入前一範本
        └── 寫入 ScriptProperties
```

### A.5 自動偵測更新

側邊欄載入後每 **30 秒**執行一次 `checkTemplateUpdate()`：

- 偵測到更新 → 顯示「正在重新掃描範本」通知 → 呼叫 `refreshSidebar()` 自動重建
- 無更新 → 靜默略過

### A.6 插入邏輯

```
applyTemplate(type)
  ├── type 格式：'主項|||子項|||細項'（以 ||| 分隔，前端傳入）
  ├── 從快取取得對應的投影片 index 陣列（slideIndexes）
