# 競品活動分析系統 — 優化建議

> 版本：v1.0　撰寫日期：2026-05-26
> 基於完整程式碼審查，涵蓋安全性、維護性、效能三個面向。

## 一、安全性問題

### ✅ 已驗證安全：前端頁籤可被 F12 繞過，但後端防線有效

**問題位置**：`permissions.html` → `updateTabsVisibility()`

**現況**：前端頁籤隱藏只修改 CSS display，技術上可透過 F12 強制顯示：
```javascript
userPermissions.allowedTabs.push('數學資料庫')
updateTabsVisibility()
// → 頁籤出現，但顯示「資料為空」
```

**實測結果（2026-05-26）**：使用部分權限帳號測試所有繞過方式，後端 `callerHasTab_()` 均正確擋下，無任何未授權資料洩漏。

**部署設定確認**：
- `Execute as`: **Me**（yayihuang@yile.com.tw）✅
- `Who has access`: **弈樂科技股份有限公司中的所有使用者** ✅

**結論：此項目無需修復，安全性符合要求。**

### ✅ 已修復：Stored XSS — innerHTML 未做完整 escape

**問題位置**：多個模組（`util.html`、`market.html`、`math.html`、`main.html`、`permissions.html`）

**修復方式**：在 `util.html` 建立統一的 `escapeHtml()` 函數，所有來自後端的字串均通過此函數處理後才插入 DOM。

**實測驗證（2026-05-26）**：
```javascript
escapeHtml('<b>測試粗體</b><img src=x onerror="alert(1)">')
// 回傳：'&lt;b&gt;測試粗體&lt;/b&gt;&lt;img src=x onerror=&quot;alert(1)&quot;&gt;'
// HTML 完全被轉成純文字，不執行 ✅
```

---

### ✅ 已確認安全：slideUrl 協定驗證

**問題位置**：`math.html`、`market.html`、`main.html` 的 `openDocument()`

**調查結果（2026-05-26）**：`slideUrl` 欄位是快取建置引擎（`自動刷新時間軸.txt`）在掃描時**自動產生**的，使用 Google Drive API 的 `file.getId()` 和 `slide.getObjectId()` 拼接，格式固定為 `https://docs.google.com/presentation/d/{id}/edit#slide=id.{slideId}`。

此欄位不從投影片內容讀取，使用者無法透過在投影片中填入惡意 URL 來影響此欄位。

**結論：此項目無需修復。**

---

### 🟢 低風險：第三方 CDN 缺少 Subresource Integrity (SRI)

**問題位置**：`main.html` 動態載入 SheetJS 與 html2canvas

**現況**：無 SRI 校驗，若 CDN 被污染，惡意程式碼會在匯出時執行。

**修復方式**：在動態載入時加入 integrity 屬性（SRI hash 可從 cdnjs.com 取得）。

## 二、維護性問題

### 🔴 高優先：兩個後端腳本有大量重複函數

**問題位置**：`競品圖表.txt` 與 `自動刷新時間軸.txt`

**重複的函數**（各維護一份，容易不同步）：

| 函數 | 影響 |
|------|------|
| `extractActivities()` | 活動解析邏輯若改一邊，另一邊不同步會造成快取與即時讀取結果不一致 |
| `processFolder()` | 同上 |
| `filterDuplicateActivities()` | 去重邏輯不一致 |
| `calculateSimilarity()` | Levenshtein 演算法完全相同 |
| `getFishGameSubcategory()` | 魚機分類規則不一致 |
| `getOtherTypeSubcategory()` | 其他類型分類不一致 |
| `logError()` | 除了 timelineName 參數外完全相同 |

**建議**：建立第三個 GAS 腳本檔案 `shared_utils.gs`（或 `Code_Shared.gs`），將上述函數移入，兩個主腳本 include 此共用層。GAS 多個 `.gs` 檔案共享同一全域作用域，無需 import/export。

---

### 🔴 高優先：前端模組跨檔案隱式依賴

**問題位置**：`market.html`、`math.html` 呼叫在 `util.html`、`timeline.html` 定義的函數

**現況**：沒有任何文件記錄這些依賴，載入順序若改變（`圖表.html` 的 `include()` 順序）系統會靜默失效。

**建議**：在每個 HTML 模組頂部加入依賴說明注釋：

```javascript
// market.html 頂部加入
// 依賴（需先載入）：util.html（buildTimelineDateHeader, buildTimelineWeekdayHeader,
//   getActivityTypeClass, getProductColor）, timeline.html（calculateActivityRowsIntelligent）
```

---

### 🟢 低優先：console.log 未清除

生產環境中保留大量 `console.log`，會在 F12 Console 留下操作記錄（包含使用者 Email、查詢條件、資料筆數）。

**建議**：
```javascript
// util.html 中已有 console.log = function(){} 的靜音
// 但這是全域不可逆操作，影響除錯
// 建議改用 debug flag
const DEBUG = false;
const log = (...args) => DEBUG && console.log(...args);
```

---

### 🟢 低優先：initializeAfterPermissions 三層巢狀回呼

**問題位置**：`main.html` 約第 2159~2230 行

**現況**：三個 `google.script.run` 以巢狀 callback 連接，縮排深達 6 層，難以閱讀。

**建議重構**：將三個呼叫分解為具名函數：
```javascript
function fetchMarketingData(onComplete) { google.script.run...getMarketingDatabaseData()... }
function fetchMathData(onComplete) { ... }
function fetchCompetitorData(onComplete) { ... }

// 主流程
checkUserPermissionsAndInitialize()
  → fetchMarketingData()
  → fetchMathData()
  → fetchCompetitorData()
  → initializeTabs()
```

---

### 🟢 低優先：setTimeout 替代 requestAnimationFrame

**問題位置**：`timeline.html`（500ms）、`util.html`（1000ms）

硬編碼的延遲在低效能裝置上可能不足，建議改用 `requestAnimationFrame` 確保 DOM 渲染完成後才執行後續操作。

## 三、效能問題

### 🟡 中優先：頁面初始化串行等待三次後端呼叫

**現況**：三個 `google.script.run` 呼叫是串行的（行銷 → 數學 → 競品活動），總等待時間 = 三次後端各自的回應時間之和。

**建議**：若三個資料來源互不依賴，可改為並行呼叫（需在前端自行實作 Promise 或計數器協調）。

---

### 🟡 中優先：數學資料庫每次切換頁籤都重新載入

**問題位置**：`main.html` → `switchTab('math')`

```javascript
// 目前行為：每次切到 math 都呼叫 loadMathDatabase()
if (competitorName === 'math') {
  if (typeof window.loadMathDatabase === 'function') {
    window.loadMathDatabase();
  }
}
```

`math.html` 的 `loadMathDatabase()` 內部有快取判斷（`if (window.mathGamesData)`），但 `initializeTabs()` 在初始化時已載入 `window.mathGamesData`，切換頁籤應直接使用快取而非重新呼叫函數。

**建議**：在 `switchTab()` 加入頁籤已初始化的旗標，避免重複呼叫。
