# 競品活動分析系統 — 使用指南維護指引

## 檔案角色

| 檔案 | 用途 |
|------|------|
| `使用指南.md` | 文字內容主檔，所有文字改這邊 |
| `_維護/使用指南_source.html` | HTML 主檔，保留圖片／影片相對路徑，文字與 MD 同步 |
| `使用指南.html` | 輸出用分享版，圖片影片全部 base64 嵌入，單一檔案可到處開 |
| `_維護/_embed.py` | 嵌入腳本，讀 source → 輸出 使用指南.html |
| `示意圖/` | 說明圖片資料夾 |
| `參考影片/` | 教學影片資料夾 |

---

## 文字內容更新流程

1. 修改 `使用指南.md`
2. 使用者確認 MD 內容沒問題時，主動問：「要同步更新 使用指南.html 嗎？」
3. 使用者說「好」→ **先問「這次的執行人是誰？」**，取得名稱後再動手
4. 同步修改 `_維護/使用指南_source.html`
5. 執行 `python _維護/_embed.py` → 輸出 `使用指南.html`

> 若使用者說「不用更新修訂紀錄」，跳過執行人步驟，直接跑 embed 輸出。

---

## 圖片／影片更新流程

圖片和影片的更新一律由 Claude 協助處理：

1. 使用者提供新檔案或說明要更換哪個
2. 將新檔案放入對應資料夾（`示意圖/` 或 `參考影片/`）
3. 若需新增從未出現的圖片或影片，先在 `_維護/使用指南_source.html` 加入對應標籤與位置，並同步更新 `使用指南.md`
4. 執行 `python _維護/_embed.py` 輸出

---

## 新增系統功能流程

當使用者說「有新功能要更新架構圖」或類似描述時：

1. **主動請對方提供最新程式碼**：說「請把最新的程式碼貼給我，或告訴我是哪個檔案，我來判斷新功能的名稱和位置。」
2. 看完程式碼後，判斷新功能屬於哪一層：
   - **競品分析系統**：使用者可直接互動的功能頁籤
   - **後端支援基礎**：資料流程、排程、儲存、權限等基礎設施
3. 同步更新架構圖：`_維護/使用指南_source.html` 的架構圖區塊 + `使用指南.md` 的 ASCII 架構圖
4. 視需要在對應章節補充功能說明
5. 依文字內容更新流程收尾（確認 → 問執行人 → 跑 embed）

---

## 重要規則

- 永遠不要直接編輯 `使用指南.html`，改了會被 embed 腳本覆蓋
- 同步 HTML 前必須先問執行人，記入修訂紀錄
- 修訂紀錄在 HTML 頁面最上方，欄位：版本、日期、執行人、修訂內容
- 正式發布前不需要記修訂紀錄
